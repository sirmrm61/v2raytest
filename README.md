# V2Ray/Xray/Hysteria2 Optimizer - نسخه مدرن

برنامه بهینه‌ساز حرفه‌ای کانفیگ‌های V2Ray/Xray/Hysteria2 با رابط کاربری مدرن، تست سرعت واقعی و پشتیبانی کامل از زبان فارسی

## ✨ ویژگی‌های جدید

### 🚀 پشتیبانی کامل از Hysteria2
- تشخیص خودکار پروتکل‌های `hysteria2://` و `hy2://`
- تست با باینری Hysteria2 برای نتایج دقیق‌تر
- گزینه رد کردن خودکار کانفیگ‌های Hysteria2
- مدیریت هوشمند خطاها برای فرآیند Hysteria

### 📈 تست سرعت واقعی
- تست سرعت دانلود برای کانفیگ‌های برتر
- تنظیم حجم تست (پیش‌فرض 10MB)
- نمایش سرعت به Mbps
- قابل فعال/غیرفعال کردن از تنظیمات

### 📊 خروجی Excel
- خروجی خودکار به فرمت Excel (.xlsx)
- شامل تمام جزئیات: تاخیر، نرخ موفقیت، امتیاز پایداری
- ستون‌های فارسی برای خوانایی بهتر
- استفاده از pandas و openpyxl

### 🎨 رابط کاربری بهبود یافته
- طراحی card-based برای تنظیمات
- نمایش زمان باقی‌مانده (ETA) و سرعت تست
- لاگ قابل کپی با کلیک راست
- عنوان جدید با emojiهای بیشتر
- رنگ‌بندی Catppuccin-inspired بهبود یافته

## ویژگی‌های اصلی

### 🎨 رابط کاربری مدرن
- طراحی dark mode با رنگ‌بندی زیبا و چشم‌نواز
- دکمه‌های سفارشی با افکت hover
- پشتیبانی کامل از RTL (راست‌چین)
- فونت‌های خوانا و مناسب برای فارسی (Vazirmatn, B Nazanin)
- نوار پیشرفت مدرن با ETA و سرعت
- جدول نتایج با استایل حرفه‌ای

### ⚡ تست شبکه بهینه
- **پشتیبانی از Xray Core** برای تست واقعی پروکسی (مانند v2rayNG)
- **پشتیبانی از Hysteria2** برای تست پروتکل‌های مدرن
- تست TCP مستقیم با socket برای سرعت بیشتر
- **نمایش همزمان Proxy Latency و TCP Latency**
- کنترل همزمانی (concurrency) قابل تنظیم
- محاسبه دقیق تاخیر (min, avg, max)
- امتیازدهی پایداری بر اساس انحراف معیار
- **Fallback هوشمند** در صورت عدم وجود باینری‌ها

### 📊 مدیریت نتایج
- رتبه‌بندی خودکار کانفیگ‌ها
- نمایش درصد موفقیت و تاخیر
- کپی سریع کانفیگ‌ها با یک کلیک
- خروجی به فرمت txt, base64, json, excel
- باز کردن فایل خروجی مستقیم از برنامه

## نصب و راه‌اندازی

### پیش‌نیازها
- Python 3.8 یا بالاتر
- کتابخانه‌های مورد نیاز (در requirements.txt)

### نصب کتابخانه‌ها
```bash
pip install -r requirements.txt
```

یا دستی:
```bash
pip install requests urllib3 pandas openpyxl
```

### اجرای برنامه
```bash
python v2ray_optimizer_modern.py
```

### تنظیم باینری‌ها (اختیاری اما توصیه شده)
برای تست دقیق‌تر، باینری‌های زیر را دانلود کنید و مسیر آن‌ها را در تنظیمات وارد کنید:

1. **Xray Core** (برای vmess, vless, trojan, ss):
   - دانلود از: https://github.com/XTLS/Xray-core/releases
   - نسخه ویندوز: `xray-windows-64.zip`
   - مسیر پیشنهادی: `./xray/xray.exe`

2. **Hysteria2** (برای hysteria2, hy2):
   - دانلود از: https://github.com/apernet/hysteria/releases
   - نسخه ویندوز: `hysteria-windows-amd64.exe`
   - مسیر پیشنهادی: `./hysteria/hysteria.exe`

## راهنمای استفاده

### 1. تب Subscription
- لینک‌های subscription خود را وارد کنید (هر خط یک URL)
- یا از فایل متنی بارگذاری کنید
- پشتیبانی از لینک‌های GitHub (تبدیل خودکار به raw)

### 2. تب تنظیمات
تب تنظیمات شامل سه بخش است:

#### 📊 تنظیمات پایه
- **نام فایل خروجی**: نام فایل‌های خروجی
- **تعداد کانفیگ برتر**: تعداد کانفیگ‌هایی که در نتایج نمایش داده می‌شوند
- **تعداد تست برای هر کانفیگ**: تعداد دفعات تست هر کانفیگ برای دقت بیشتر
- **تعداد همزمانی**: تعداد تست‌های همزمان (بیشتر = سریع‌تر اما مصرف CPU بیشتر)
- **Timeout هر تست**: زمان انتظار برای هر تست (ثانیه)
- **حداکثر تاخیر مجاز**: کانفیگ‌هایی با تاخیر بیشتر از این مقدار نادیده گرفته می‌شوند

#### 🔧 مسیر باینری‌ها
- **مسیر Xray/V2Ray**: مسیر باینری Xray برای تست پروتکل‌های vmess, vless, trojan, ss
- **مسیر Hysteria2**: مسیر باینری Hysteria2 برای تست پروتکل‌های hysteria2, hy2
- **رد کردن Hysteria2**: اگر فعال باشد، کانفیگ‌های Hysteria2 را رد می‌کند

#### 🚀 تنظیمات پیشرفته
- **زمان‌بندی خودکار**: تست خودکار در بازه‌های زمانی مشخص
- **تست سرعت واقعی**: تست دانلود برای کانفیگ‌های برتر
- **حجم تست سرعت**: حجم فایل دانلود برای تست سرعت (MB)

### 3. تب اجرای تست
- دکمه "🚀 شروع تست و بهینه‌سازی" را بزنید
- پیشرفت تست را در نوار پیشرفت، ETA و سرعت تست مشاهده کنید
- لاگ زنده را برای جزئیات تست بررسی کنید
- با کلیک راست روی لاگ می‌توانید متن را کپی کنید
- پس از اتمام، به تب نتایج بروید

### 4. تب نتایج
- کانفیگ‌ها بر اساس پایداری و سرعت رتبه‌بندی شده‌اند
- با کلیک روی ستون "📋" هر کانفیگ را کپی کنید
- دکمه "کپی همه" برای کپی تمام کانفیگ‌ها
- دکمه "باز کردن فایل خروجی" برای مشاهده فایل‌های خروجی

## فایل‌های خروجی

پس از اتمام تست، فایل‌های زیر ایجاد می‌شوند:

- `{output_name}.txt`: کانفیگ‌ها به فرمت base64 (برای v2rayNG)
- `{output_name}.base64`: کانفیگ‌ها به فرمت متنی ساده
- `{output_name}_results.json`: نتایج دقیق به فرمت JSON
- `{output_name}_results.xlsx`: نتایج به فرمت Excel با ستون‌های فارسی

## پروتکل‌های پشتیبانی شده

- `vmess://` - با پشتیبانی Xray
- `vless://` - با پشتیبانی Xray
- `trojan://` - با پشتیبانی Xray
- `ss://` (Shadowsocks) - با پشتیبانی Xray
- `hysteria2://` - با پشتیبانی Hysteria2
- `hy2://` - با پشتیبانی Hysteria2

## بهینه‌سازی‌های انجام شده

### تست شبکه
- استفاده از socket مستقیم به جای asyncio.open_connection برای سرعت بیشتر
- ThreadPoolExecutor برای مدیریت بهتر threadها
- کاهش overhead در تست‌های متعدد
- پشتیبانی از Hysteria2 با باینری اختصاصی
- مدیریت هوشمند خطاها برای فرآیندهای خارجی

### رابط کاربری
- طراحی custom buttons برای ظاهر مدرن‌تر
- استفاده از رنگ‌های Catppuccin-inspired بهبود یافته
- بهبود خوانایی با فونت‌های مناسب (Vazirmatn, B Nazanin)
- انیمیشن‌های hover برای تعامل بهتر
- طراحی card-based برای تنظیمات
- نمایش ETA و سرعت تست در زمان واقعی

### عملکرد
- پیش‌فرض‌های بهینه برای تست سریع (5 تست per config, 50 concurrency)
- timeout کاهش یافته به 3 ثانیه برای سرعت بیشتر
- حذف کانفیگ‌های تکراری قبل از تست
- Early stopping هوشمند وقتی تعداد کانفیگ‌های خوب به top_n برسد
- تست سرعت واقعی برای کانفیگ‌های برتر

## تنظیمات پیش‌فرض

```json
{
  "subscriptions": [],
  "output_name": "best-configs",

  "tests_per_config": 5,

  "auto_schedule_enabled": false,
  "auto_schedule_interval": 60,
  "max_allowed_latency": 500,

  "speed_test_size": 10

```

## نکات

- برای تست‌های دقیق‌تر، تعداد تست per config را افزایش دهید
- برای سرعت بیشتر، concurrency را افزایش دهید (مراقب مصرف CPU)
- timeout کمتر = تست سریع‌تر اما ممکن است کانفیگ‌های کند را از دست بدهید
- تنظیمات در فایل config.json ذخیره می‌شوند

## مشکلات رایج

**برنامه اجرا نمی‌شود**: مطمئن شوید Python 3.8+ نصب است و requests نصب شده است.

**تست‌ها خیلی کند هستند**: concurrency را افزایش دهید یا timeout را کاهش دهید.

**هیچ کانفیگی پیدا نشد**: لینک subscription را بررسی کنید یا از فایل متنی مستقیم استفاده کنید.
